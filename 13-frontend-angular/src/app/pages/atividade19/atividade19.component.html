Estratégias de Renderização e Carregamento Diferido

PAINEL DE INICIALIZAÇÃO

O monitor de hardware abaixo é um componente "pesado" que não faz parte do bundle inicial.

@defer (when isSystemReady()) { } @loading (minimum 1s) {
BAIXANDO MÓDULO DE TELEMETRIA...
} @placeholder {

MÓDULO DE HARDWARE OFFLINE. DISPARE O BOOT PARA CARREGAR.

} @error {

ERRO CRÍTICO AO CARREGAR MÓDULO DE PERFORMANCE.

}

ESPECIFICAÇÃO TÉCNICA

  • @defer (when): Carregamento assíncrono disparado por estado reativo.
  • OnPush Strategy: Componente filho ignora ciclos de change detection globais.
  • Bundle Splitting: O componente pesado é separado em um chunk JS distinto.
