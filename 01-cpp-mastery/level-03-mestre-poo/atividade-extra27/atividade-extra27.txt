## 🎯 Estudo de Caso
Análise técnica do cenário de negócio para o componente atividade-extra27.txt.

ESTUDO DE CASO: SISTEMA DE GESTÃO DE CONTAINERS (LOGÍSTICA 4.0)
------------------------------------------------------------------------------
DOMÍNIO: Logística / Transporte Marítimo e Aéreo
ATIVIDADE: Extra 27
CONCEITO NOVO: Class Templates (Templates de Classe)
STATUS: Padrão de Engenharia de Elite Aplicado

1. O CENÁRIO
Uma transportadora internacional lida com diferentes tipos de unidades de carga. 
Alguns containers levam apenas 'Cargas Líquidas' (medidas em double), outros 
levam 'Lotes de Peças' (medidos em int), e alguns levam 'Objetos Complexos' 
(como máquinas descritas por structs).

2. A NECESSIDADE TÉCNICA
Até agora, se quiséssemos uma classe 'Container', teríamos que fixar o tipo 
de dado que ela guarda. Para cada novo tipo de carga, precisaríamos criar uma 
nova classe (ContainerInt, ContainerDouble, etc).

O TEMPLATE DE CLASSE permite criar um "Container Genérico". Você define a 
estrutura da classe uma única vez e, na hora de criar o objeto, diz ao C++ 
o que ele vai carregar.

3. 🧪 ## 🧪 ANÁLISE DE FALHA CRÍTICA (NÍVEL ARQUITETURA GENÉRICA)
- RISCO DE INCOMPATIBILIDADE DE INTERFACE: Se o template realizar uma operação 
  (ex: cout << conteudo) e o tipo 'T' passado não suportar essa operação (ex: 
  uma struct sem sobrecarga de <<), o sistema falhará com erros de compilação 
  extensos.
- RISCO DE HEADER-ONLY DEPENDENCY: Diferente de classes comuns, templates de 
  classe devem ter sua implementação completa no arquivo de cabeçalho (.h), o 
  que pode aumentar o tempo de compilação em projetos gigantescos.
- RISCO DE OVERHEAD DE CÓPIA: Se o tipo 'T' for um objeto pesado e o template 
  não usar referências constantes (const T&), a performance do sistema de 
  logística será severamente degradada por cópias inúteis na memória.

4. REQUISITOS DO SISTEMA
- Implementação de Class Template 'UnidadeDeCarga<T>'.
- Proteção de performance via Pass-by-Reference (Fantasma do CPU).
- Interface ANSI para telemetria de manifesto de carga.
