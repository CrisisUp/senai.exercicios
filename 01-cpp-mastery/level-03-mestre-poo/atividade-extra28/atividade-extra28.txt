Implementação de padrões de engenharia de elite para garantir escalabilidade e robustez.

## 🎯 Estudo de Caso
Análise técnica do cenário de negócio para o componente atividade-extra28.txt.

ESTUDO DE CASO: CHECKOUT DINÂMICO DE SUPERMERCADO (BUSCA POR SKU)
------------------------------------------------------------------------------
DOMÍNIO: Varejo / Automação Comercial
ATIVIDADE: Extra 28
CONCEITO NOVO: STL std::map (Dicionários Chave-Valor)
STATUS: Padrão de Engenharia de Elite Aplicado

1. O CENÁRIO
Um supermercado possui milhares de produtos. No momento do checkout, o operador 
passa o código de barras (SKU) e o sistema precisa retornar o preço e o nome 
instantaneamente. 

2. A NECESSIDADE TÉCNICA
Até agora, usamos 'std::vector'. Para achar um produto em um vetor de 10.000 itens, 
o computador teria que olhar item por item (Busca Linear O(n)). Isso é lento.

O STL std::map (Mapa) resolve isso usando uma estrutura de Árvore Binária:
- Chave-Valor: Associação de uma CHAVE única a um VALOR.
- Performance: Busca logarítmica O(log n), garantindo escalabilidade para 
  grandes inventários.

3. 🧪 ## 🧪 ANÁLISE DE FALHA CRÍTICA (NÍVEL ESTRUTURA DE DADOS)
- RISCO DE BUSCA INEXISTENTE: Acessar uma chave inexistente via `map[chave]` cria 
  automaticamente um item vazio no mapa. Isso pode "poluir" o banco de dados com 
  lixo. Solução: Usar sempre `.find()` ou `.at()`.
- RISCO DE COMPLEXIDADE DE INSERÇÃO: Embora a busca seja rápida, a inserção no 
  mapa exige o rebalanceamento da árvore (Red-Black Tree), o que é mais custoso 
  que um simples `push_back` no vector.
- RISCO DE OVERHEAD DE MEMÓRIA: Cada nó de um `std::map` possui três ponteiros 
  extras (pai, esquerda, direita) e um bit de cor. Para dados muito pequenos, 
  o consumo de RAM pode dobrar em relação a um array contíguo.

4. REQUISITOS DO SISTEMA
- Motor de Busca 'SistemaPrecos' baseado em Árvore Binária (std::map).
- Implementação de 'Guardião Financeiro' (Long Long Cents).
- Proteção de acesso via iteradores seguros.
- Interface ANSI para feedback de checkout.
