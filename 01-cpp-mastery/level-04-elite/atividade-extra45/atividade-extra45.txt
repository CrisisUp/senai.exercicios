## 🛠️ Necessidade Técnica
Implementação de padrões de engenharia de elite para garantir escalabilidade e robustez.

## 🎯 Estudo de Caso
Análise técnica do cenário de negócio para o componente atividade-extra45.txt.

ESTUDO DE CASO: PILOTAGEM DE DRONE EM TEMPO REAL (G-PILOT)
------------------------------------------------------------------------------
DOMÍNIO: Robótica / Sistemas de Inspeção / Jogos
ATIVIDADE: Extra 45
CONCEITO NOVO: Input Não-Bloqueante (Raw Mode Terminal e fcntl)
STATUS: Padrão de Engenharia de Elite Aplicado

1. O CENÁRIO
Um drone de inspeção industrial está dentro de uma tubulação perigosa. O piloto 
precisa desviar de obstáculos rapidamente. Se o software usar 'std::cin', o 
drone ficará parado esperando o piloto digitar 'W' e apertar 'Enter'. Nesse 
tempo, o drone bateria na parede.

2. A NECESSIDADE TÉCNICA (Quebrando a barreira do Enter)
Por padrão, o terminal trabalha em "Cooked Mode": ele guarda o que você digita 
em um buffer e só entrega ao programa quando você aperta Enter.
Para um jogo Senior, precisamos do RAW MODE:
- Termios.h: Permite desligar o "Echo" e o "Canonical Mode".
- Fcntl.h: Configura o descritor de arquivo do teclado como "Non-Blocking".

3. 🧪 ## 🧪 ANÁLISE DE FALHA CRÍTICA (NÍVEL PROGRAMAÇÃO DE SISTEMAS)
- RISCO DE ZOMBIE TERMINAL: Se o programa sofrer um Crash antes de executar o 
  destrutor da classe 'InputManager', o terminal do usuário ficará "quebrado" 
  (sem eco e sem enter), exigindo o comando `reset` para voltar ao normal. 
  Solução: Captura de Sinais (Signals).
- RISCO DE INPUT LAG (O_NONBLOCK): Se o loop de polling for lento demais, 
  teclas pressionadas podem ser perdidas. Se for rápido demais, o drone 
  "teletransportará" em vez de se mover suavemente.
- RISCO DE INCOMPATIBILIDADE S.O.: O código `termios.h` é nativo de sistemas 
  Unix (Linux/macOS). Tentar rodar este executável no Windows CMD resultará em 
  falha total de compilação.

4. REQUISITOS DO SISTEMA
- Gestão de Terminal via 'termios' (Low-level System Call).
- Mecânica de Input Assíncrona e Não-bloqueante.
- Proteção de Estado via Padrão RAII.
- Interface ANSI para telemetria de voo em tempo real.
