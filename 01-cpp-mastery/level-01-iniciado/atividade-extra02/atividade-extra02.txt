FOLHA DE ATIVIDADE - Lógica de Programação e Algoritmos
SA 2 – Desafio 1 - ORGANIZAÇÃO DE DADOS: VETORES

Objetivo da atividade:
Desenvolver a capacidade de utilizar vetores, aplicar estruturas de repetição e condicionais para resolver problemas com a linguagem C++. Nota: 12

Contexto:
Você trabalha como estagiário em uma empresa que desenvolve soluções digitais 
para o agronegócio. Um cliente rural deseja um programa simples que o ajude a monitorar a irrigação dos setores de sua fazenda.
Você já criou um sistema que registra a quantidade de água usada em cada setor. Agora, o cliente solicitou um complemento: o programa deve registrar a quantidade de ração oferecida ao gado em cada curral da fazendo, e identificar os currais 
que receberam quantidade insuficiente de ração.

Orientações:

1. Solicite ao usuário:
• A quantidade de currais da fazenda (numCurrais)
• A quantidade mínima recomendada de ração por curral (minimoRacao)

2. Crie um vetor para armazenar a quantidade de ração oferecida em cada curral (racaoCurrais)

3. Solicite ao usuário que informe a quantidade de ração fornecida em cada curral.

4. Percorra o vetor usando um laço de repetição e, com condicional simples, verifique:
• Quais currais receberam menos ração que o mínimo recomendado.

5. Exiba no console:
• A quantidade de ração de cada curral

Pontos de atenção:
Envie o arquivo .cpp criado. Teste o .exe antes de enviar.
• Use vetores e laço for para percorrer os dados.
• Use if para verificar se a ração está abaixo do mínimo.
• Exiba os resultados com mensagens claras no console.
• Comente seu código explicando cada etapa.

Capacidades que você vai desenvolver:
• Criar estruturas de repetição para executar um conjunto de instruções várias vezes;
• Utilizar estruturas condicionais para executar instruções com base em uma condição;
• Utilizar lógica de programação para a resolução de problemas;
• Aplicar técnicas de código limpo (clean code).

Você será avaliado(a) por:
• Enviar o arquivo .cpp criado, e lembrou-se de testar o arquivo .exe criado 
antes de enviar a atividade;
• Utilizar variáveis adequadas para os dados informados (inteiros e float ou double para valores monetários);
• Usar operadores aritméticos para calcular a quantidade a ser reposta e o custo total;
• Utilizar operadores lógicos e estruturas de decisão (if) para verificar a necessidade de reposição;
• Exibir mensagens claras no console com cout;
• Comentar seu código para explicar cada etapa.

Análise de Falha Crítica:
Risco de Input Failure: O sistema atual utiliza cin diretamente para capturar dados numéricos (int e double). Caso o usuário digite letras em um campo numérico, o cin entra em estado de erro (failbit), para de ler dados e entra em um loop infinito ou pula as próximas entradas, travando a lógica de negócio. Em sistemas reais, isso pode causar corrupção de memória ou negação de serviço.
