# 📘 Atividade 01: O Primeiro Molde (Nível 01)
STATUS: Nível 01 (Escultor) - Trilha POO Mastery

## 🎯 Estudo de Caso
Imagine que você é o engenheiro-chefe da "SkyCargo Nexus". Até ontem, você gerenciava os drones usando variáveis soltas como `drone1_bateria`, `drone1_modelo`, etc. Isso é um pesadelo de organização. Você aprenderá a criar a classe `Drone`, um "projeto de engenharia" que define quais dados e ações cada drone terá, permitindo criar milhares de instâncias organizadas.

## 🛠️ Necessidade Técnica
Uma **Classe** é o molde (blueprint). Um **Objeto** é a instância física criada a partir desse molde. Atributos são as características (variáveis) e Métodos são os comportamentos (funções).

### Estrutura Base (C++):
```cpp
class Drone {
public: // Acesso público por enquanto
    string modelo;
    int bateria;

    void decolar() {
        cout << "Drone " << modelo << " decolando..." << endl;
    }
};
```

## 🚀 Missão
1.  Crie a pasta `labs/drone-factory` e um arquivo `main.cpp`.
2.  Defina a classe `Drone` com atributos `modelo` e `bateria`.
3.  Crie um método `verStatus()` que imprima as informações do drone.
4.  No `main()`, instancie dois objetos: `droneA` e `droneB`.
5.  Atribua valores diferentes para eles e chame os métodos.

## 📋 Requisitos de Elite
- A classe deve representar uma entidade do mundo real.
- Uso correto da sintaxe de membros (`objeto.atributo`).
- Saída limpa no console usando cores ANSI (se possível).
