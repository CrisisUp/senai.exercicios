# 📘 Atividade 02: O Ciclo Vital (Construtores e Destrutores)
STATUS: Nível 02 (Escultor) - Trilha POO Mastery

## 🎯 Estudo de Caso
Na "SkyCargo Nexus", um drone não pode nascer "vazio" ou sem nome. Além disso, quando um drone é aposentado (deletado da memória), ele deve emitir um sinal de log final. Você aprenderá a usar o **Construtor** (o parto do objeto) e o **Destrutor** (o encerramento), garantindo que a memória seja gerenciada com precisão cirúrgica.

## 🛠️ Necessidade Técnica
- **Construtor:** Método especial chamado automaticamente no nascimento do objeto. Serve para inicializar atributos.
- **Destrutor:** Método chamado no encerramento (ao sair do escopo ou deletar). Serve para limpar lixo ou fechar conexões.
- **`this->`**: O ponteiro que aponta para o próprio objeto (auto-referência).

### Exemplo (C++):
```cpp
class Drone {
public:
    Drone(string m) { // Construtor
        this->modelo = m;
    }
    ~Drone() { // Destrutor
        cout << "Limpando hangar do " << modelo << endl;
    }
};
```

## 🚀 Missão
1.  Na classe `Drone`, adicione um construtor que peça o `modelo` e a `bateria`.
2.  Adicione um destrutor que imprima "Drone [MODELO] DESATIVADO".
3.  No `main()`, crie um objeto dentro de um bloco `{ }` para ver o destrutor rodando automaticamente ao final do bloco.
4.  Use o ponteiro `this` para diferenciar atributos de parâmetros no construtor.

## 📋 Requisitos de Elite
- Inicialização obrigatória via construtor.
- Demonstração do ciclo de vida (nascimento e morte) no terminal.
- Compreensão de como o escopo afeta a vida do objeto.
