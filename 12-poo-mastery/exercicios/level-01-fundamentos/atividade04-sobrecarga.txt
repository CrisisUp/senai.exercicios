# 📘 Atividade 04: Versatilidade de Comando (Sobrecarga de Métodos)
STATUS: Nível 04 (Escultor) - Trilha POO Mastery

## 🎯 Estudo de Caso
Um drone de elite da "SkyCargo Nexus" deve ser flexível. Às vezes ele decola em modo padrão, outras vezes precisa de uma decolagem rápida para emergências, ou até uma decolagem direcionada para coordenadas específicas. Em vez de criar nomes confusos como `decolarPadrao()` e `decolarCoordenadas()`, você aprenderá a usar a **Sobrecarga (Overloading)**: o mesmo nome de método com diferentes "assinaturas" (parâmetros).

## 🛠️ Necessidade Técnica
A Sobrecarga permite que uma classe tenha vários métodos com o mesmo nome, desde que a lista de parâmetros seja diferente (tipo ou quantidade). O compilador decide qual versão chamar com base nos argumentos que você passa.

### Exemplo (C++):
```cpp
void decolar() { /* modo padrão */ }
void decolar(float altitude) { /* decola até X metros */ }
void decolar(string destino) { /* decola rumo a local */ }
```

## 🚀 Missão
1.  Na classe `Drone`, mantenha o método `decolar()` original.
2.  Crie uma sobrecarga `decolar(int velocidade)` que imprima uma mensagem diferenciada (ex: "Decolagem Turbo a [V] km/h").
3.  Crie outra sobrecarga `decolar(double lat, double lon)` para decolagem direcionada.
4.  No `main()`, chame as três versões do método para o mesmo objeto e observe como o C++ diferencia os comandos.

## 📋 Requisitos de Elite
- Implementar pelo menos 3 versões do mesmo método.
- Usar tipos de dados diferentes nos parâmetros (int, double, string).
- Compreender que o tipo de retorno não conta para diferenciar a sobrecarga (apenas os parâmetros).
