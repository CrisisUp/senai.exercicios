# 📘 Atividade 22: Especializando o Voo (Sobrescrita - Overriding)
STATUS: Nível 22 (Arquiteto de Linhagens) - Trilha POO Mastery

## 🎯 Estudo de Caso
Um drone de **Vigilância** não decola como um drone de carga. Ele precisa ser silencioso e furtivo. Na "SkyCargo Nexus", queremos que o comando `decolar()` funcione para todos, mas que cada classe filha possa redefinir como esse comando é executado. Você aprenderá a **Sobrescrita (Overriding)**, permitindo que a filha substitua a lógica do pai por uma mais específica.

## 🛠️ Necessidade Técnica
- **`virtual`**: Indica na classe pai que o método pode ser sobrescrito pelas filhas.
- **`override`**: Garante que você está realmente substituindo um método virtual do pai (segurança de compilação).

### Exemplo (C++):
```cpp
class Drone {
public:
    virtual void decolar() { cout << "Decolagem Padrão" << endl; }
};

class DroneVigilancia : public Drone {
public:
    void decolar() override { cout << "Decolagem Furtiva Ativada" << endl; }
};
```

## 🚀 Missão
1.  Na classe `Drone`, marque o método `decolar()` como `virtual`.
2.  Crie a classe `DroneVigilancia` herdando de `Drone`.
3.  Sobrescreva o método `decolar()` na classe filha para imprimir uma mensagem personalizada.
4.  No `main()`, crie um objeto `DroneVigilancia` e chame `decolar()`. Observe que o C++ executa a versão da filha.

## 📋 Requisitos de Elite
- Uso mandatório da palavra-chave `virtual` no pai e `override` na filha.
- Demonstrar a mudança de comportamento sem mudar a assinatura do método.
- Manter o contrato de interface (o nome do método permanece o mesmo).
