# 📘 Atividade 11: O Observador Imutável (Const-Correctness)
STATUS: Nível 11 (Guardião) - Trilha POO Mastery

## 🎯 Estudo de Caso
Um bug sutil na "SkyCargo Nexus" permitiu que um método de "leitura de telemetria" alterasse acidentalmente o ID de voo de um drone. Na Engenharia de Elite, se um método apenas "lê" dados, ele deve ser marcado como **const**. Você aprenderá a blindar seus métodos e referências para garantir que operações de consulta nunca modifiquem o estado do objeto.

## 🛠️ Necessidade Técnica
- **Métodos `const`**: Garantem que o método não alterará nenhum atributo da classe.
- **Referências `const`**: Permitem passar objetos grandes sem cópia (performance) e sem risco de alteração (segurança).

### Exemplo (C++):
```cpp
// Este método PROMETE não mudar nada no Drone
void exibirRelatorio() const {
    // bateria = 100; // ERRO DE COMPILAÇÃO AQUI!
    cout << bateria << endl;
}

// Passagem de parâmetro de elite
void processar(const Drone& d) { ... }
```

## 🚀 Missão
1.  Na sua classe `Drone`, marque todos os Getters e o método `verStatus()` como **const**.
2.  Crie uma função externa `analisarDrone(const Drone& d)` que chame os métodos const.
3.  Tente alterar um atributo dentro de um método `const` e observe o compilador agindo como seu primeiro nível de defesa.
4.  Entenda que o uso de `const` permite otimizações agressivas do compilador.

## 📋 Requisitos de Elite
- Aplicação de `const` em todos os métodos de leitura.
- Passagem de objetos por referência constante (`const &`) para evitar overhead de memória.
- Código livre de efeitos colaterais indesejados.
