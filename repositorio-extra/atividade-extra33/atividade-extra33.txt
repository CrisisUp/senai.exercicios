DESAFIO LENDÁRIO: NÚCLEO DE INTELIGÊNCIA BANCÁRIA (G-CENTRAL SOC)
------------------------------------------------------------------------------
DOMÍNIO: Segurança Nacional / Fintech de Alta Disponibilidade
ATIVIDADE: Extra 33 (O ÁPICE DO APRENDIZADO)
PADRÕES INTEGRADOS: Singleton, Observer, Factory, Templates, STL, Exceções.
STATUS: Padrão de Engenharia de Elite Aplicado

1. O CENÁRIO
O Banco Central (BACEN) opera o G-CENTRAL, um sistema de monitoramento de risco 
que analisa cada transferência bilionária no país. O sistema deve ser imune a 
erros de entrada, alertar a diretoria em tempo real e mobilizar agentes de 
resposta cibernética instantaneamente.

2. A MISSÃO TÉCNICA (A Trindade Lendária)
Este projeto é a consolidação final da sua maestria em C++. Você deve unir:
- SINGLETON (O Escudo): Gestão única e blindada de IOStream.
- OBSERVER (A Vigilância): Notificação desacoplada de telemetria de risco.
- FACTORY (A Resposta): Instanciação dinâmica de contramedidas de segurança.

3. 🧪 ANÁLISE DE FALHA CRÍTICA (NÍVEL ARQUITETURA LENDÁRIA)
- RISCO DE CONTENTAMENTO DE RECURSOS: Em sistemas de alta carga, o Singleton 
  gerenciando o IO pode se tornar um gargalo para as notificações do Observer.
- RISCO DE MEMORY FRAGMENTATION (FACTORY): A criação e destruição incessante de 
  objetos de defesa (IAgente) na Heap exige uma estratégia de limpeza rigorosa 
  para evitar vazamentos em turnos de 24h/7.
- RISCO DE OVERFLOW DE AUDITORIA: O uso de 'std::stack' para logs em memória 
  consome a RAM linearmente. Em auditorias reais, os dados devem ser movidos 
  para o disco (Persistência) periodicamente para liberar a memória.

4. REQUISITOS DO SISTEMA
- Integração de Múltiplos Design Patterns (Trindade da Engenharia).
- Implementação de 'Guardião Financeiro' (Long Long Cents).
- Proteção de Performance (CPU Ghost) e Robustez (Exceções).
- Interface de Comando de Operações (SOC) ANSI.
