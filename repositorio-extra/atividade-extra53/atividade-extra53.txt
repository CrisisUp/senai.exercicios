# 📘 Atividade Extra 53: Herança Simples (Gestão de Frota Logística)

## 🎯 Estudo de Caso
A transportadora "LogiSpeed" opera uma frota variada de veículos. Eles perceberam que todos os seus veículos (Caminhões, Vans, Carretas) compartilham características comuns (Placa, Marca, Modelo), mas cada um tem suas particularidades. Em vez de duplicar o código para cada tipo de veículo, eles querem uma estrutura onde as características comuns sejam herdadas de uma base.

## 🛠️ Necessidade Técnica
A **Herança (Inheritance)** é o pilar da Programação Orientada a Objetos que estabelece a relação "**É-Um**" (Is-A).
*   **Classe Base (Pai):** Contém os atributos e comportamentos genéricos.
*   **Classe Derivada (Filho):** Herda tudo da base e adiciona suas especializações.
*   **Vantagem:** Reuso massivo de código e facilidade de manutenção (se a marca mudar de `string` para um `id`, mudamos apenas na base).

## 📋 Requisitos (Arquitetura Modular)
1. Criar a classe base `Veiculo` (Placa, Marca, Odômetro).
2. Criar a classe derivada `Caminhao` que herda de `Veiculo` e adiciona `capacidadeCarga`.
3. Demonstrar o uso de **Modificadores de Acesso** (`protected`) para permitir que as classes filhas acessem atributos da base com segurança.
4. Criar `Frota.h`, `Frota.cpp` e `atividade-extra53-frota.cpp`.
