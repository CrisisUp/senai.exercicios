DOMÍNIO: Finanças Internacionais / Investimentos
ATIVIDADE: Extra 26
CONCEITO NOVO: Function Templates (Programação Genérica)
STATUS: Padrão de Engenharia de Elite Aplicado

1. O CENÁRIO
Uma corretora de valores opera com diferentes tipos de precisão financeira. 
Alguns cálculos de conversão de moeda usam 'int' (para moedas sem decimais 
ou centavos), outros usam 'float' para rapidez, e os mais críticos usam 
'double' ou 'long double' para precisão bancária total.

2. A NECESSIDADE TÉCNICA
Até agora, se quiséssemos criar uma função 'multiplicarPorTaxa', teríamos que 
criar três versões: uma para int, uma para float e uma para double. Isso gera 
duplicação de código.

Os TEMPLATES permitem criar um "molde" de função. Você escreve o código uma 
única vez usando um tipo genérico 'T', e o compilador C++ gera automaticamente 
a versão correta para cada tipo de dado que você usar.

3. 🧪 ANÁLISE DE FALHA CRÍTICA (NÍVEL ENGENHARIA GENÉRICA)
- RISCO DE CODE BLOAT (INCHAÇO DO BINÁRIO): Para cada tipo diferente usado (int, 
  float, double, long long), o compilador gera uma cópia física da função no 
  binário. Se usado em excesso com muitos tipos, o executável pode crescer 
  desnecessariamente.
- RISCO DE ERROS DE COMPILAÇÃO CRÍPTICOS: Erros em templates costumam gerar 
  centenas de linhas de log (Template Backtrace). Um simples esquecimento de 
  operador (ex: tentar somar uma string num template de soma) pode ser difícil 
  de depurar.
- RISCO DE PERDA DE PRECISÃO IMPLÍCITA: Se o template não usar 'decltype' ou 
  regras de promoção de tipos, converter um 'long double' para 'float' via 
  template causará perda silenciosa de dados.

4. REQUISITOS DO SISTEMA
- Template de Função Multi-Tipo (C++11/14 semantics).
- Implementação de segurança de tipos via Dedução Automática.
- Interface ANSI para visualização de precisão monetária.
