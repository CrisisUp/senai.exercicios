# 📘 Atividade Extra 19: Gestão de Inventário de Alta Performance

## 🏢 Estudo de Caso: "Eletrônica Tech-Flow"
A Tech-Flow é uma distribuidora de componentes eletrônicos que lida com milhares de itens. O sistema antigo usava vetores de tamanho fixo (arrays), o que causava travamentos quando o estoque crescia ou desperdício de memória quando estava vazio.

## 🛠️ Necessidade Técnica
Para sistemas modernos, não podemos prever o número exato de itens. Precisamos de **Coleções Dinâmicas** que cresçam conforme a necessidade. Além disso, a busca manual em listas desordenadas é lenta; precisamos de **Algoritmos de Ordenação** eficientes.

## 🎯 Requisitos da Aplicação
1.  **Classe Produto:** Deve conter SKU (código), Nome, Quantidade e Preço.
2.  **Uso de std::vector:** Armazenar os produtos sem limite fixo de tamanho.
3.  **Adição Dinâmica:** Permitir que o usuário adicione quantos produtos desejar.
4.  **Ordenação Profissional:** O sistema deve oferecer a opção de ordenar o estoque por PREÇO ou por NOME (A-Z) usando `std::sort`.
5.  **Interface ANSI:** Feedback visual para itens com estoque baixo (quantidade < 5).

## 🚀 Novos Conceitos C++
-   `std::vector`: O container mais importante do C++.
-   `push_back()`: Inserção dinâmica no final da lista.
-   `std::sort`: Algoritmo QuickSort/Introsort altamente otimizado.
-   **Funções Lambda (`[](){}`):** Passar critérios de ordenação de forma elegante e concisa.
