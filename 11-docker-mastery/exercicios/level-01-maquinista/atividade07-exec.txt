# 📘 Atividade 07: Manutenção de Emergência (Docker Exec)
STATUS: Nível 07 (Maquinista) - Trilha Docker Mastery

## 🎯 Estudo de Caso
O servidor web `nexus-web` (Nginx) está rodando em segundo plano, mas o Comandante Nexus percebeu que a página inicial está com um erro ortográfico. Você não quer derrubar o container para subir um novo; você precisa entrar no container "em voo" para fazer o ajuste fino. Você aprenderá a usar o comando de elite para operar dentro de containers ativos.

## 🛠️ Necessidade Técnica
O comando `run` cria um container NOVO. O comando `exec` entra em um container que JÁ ESTÁ RODANDO. Essa é a ferramenta definitiva para depuração e manutenção rápida.

### Comandos Chave:
1.  **`docker exec -it <id-ou-nome> <comando>`**: Executa um processo dentro de um container ativo.
    - Ex: `docker exec -it nexus-web bash` (Entra no terminal do container).
2.  **`docker exec <id-ou-nome> ls /etc`**: Roda um comando e sai imediatamente (sem modo interativo).

## 🚀 Missão
1.  Inicie um container Nginx em background: 
    `docker run -d --name maintenance-lab nginx`.
2.  Tente "espiar" a configuração do servidor sem entrar nele:
    `docker exec maintenance-lab ls /etc/nginx`.
3.  **O Desafio de Elite:** Agora, entre no container em modo interativo:
    `docker exec -it maintenance-lab bash`.
4.  Dentro do container, veja quem você é: `whoami`.
5.  Veja os processos rodando no "hangar virtual": `ps aux` (pode ser que precise instalar o procps, mas tente o comando).
6.  Saia do container (`exit`).
7.  Verifique: o container `maintenance-lab` ainda está rodando? `docker ps`. (Sim, o `exec` não encerra o container ao sair).
8.  Limpeza: `docker stop maintenance-lab` e `docker rm maintenance-lab`.

## 📋 Requisitos de Elite
- Diferenciar claramente `docker run` (nascimento) de `docker exec` (visita).
- Utilizar o `exec` para auditoria rápida de arquivos internos.
- Compreender que sair de um shell aberto via `exec` não interrompe o serviço principal do container.
